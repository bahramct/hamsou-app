# ابزارهای توسعه (Dev Tools)

## توضیح
این ابزارها فقط برای **توسعه** (Development) طراحی شده‌اند و در **Production** غیرفعال خواهند شد.

## مهم: حذف خودکار داده‌های قبلی
در حالت توسعه، **همیشه قبل از تولید داده‌های جدید، داده‌های قبلی پاک می‌شوند**. این باعث می‌شود:
- داده‌های قدیمی و کش شده پاک شوند
- همیشه داده‌های جدید و تمیز داشته باشید
- تست‌ها قابل تکرار باشند

## هدف
برای تست گزارش هفتگی و تحلیل AI، به داده‌های تستی نیاز داریم. این ابزارها به ما اجازه می‌دهند:
- داده‌های تستی با تعداد روزهای مختلف تولید کنیم
- داده‌های موجود را ببینیم
- داده‌ها را حذف کنیم

## نحوه استفاده

### 1. مشاهده ابزارهای توسعه
- به صفحه Dashboard بروید
- پایین صفحه یک کارت با عنوان "ابزارهای توسعه" و badge "DEV ONLY" را خواهید دید
- این کارت فقط در محیط development نمایش داده می‌شود

### 2. تولید داده تستی
- **اولین بار**: اگر داده‌ای ندارید، سه دکمه را خواهید دید:
  - **7 روز (1 هفته)**: برای تست سریع
  - **14 روز (2 هفته)**: برای تست کامل‌تر
  - **28 روز (4 هفته)**: برای تست گسترده
- **دفعات بعد**: اگر قبلاً داده تولید کرده‌اید:
  - دکمه‌ها به شکل "1 هفته جدید"، "2 هفته جدید"، "4 هفته جدید" نمایش داده می‌شوند
  - با کلیک روی هر دکمه، داده‌های قبلی پاک می‌شوند و داده‌های جدید تولید می‌شوند

### 3. پاک کردن داده‌ها
- در بالای کارت، دکمه "پاک کردن" وجود دارد
- این دکمه فقط زمانی نمایش داده می‌شود که داده‌ای موجود باشد
- با کلیک روی آن، همه داده‌ها حذف می‌شوند

### 4. مشاهده اطلاعات هفته‌ها
- بعد از تولید داده، لیست هفته‌ها نمایش داده می‌شود
- هر هفته نشان می‌دهد:
  - تعداد روزها
  - تعداد تکمیل شده
  - نرخ تکمیل (با رنگ: سبز > 70%， زرد 50-70%， قرمز < 50%)

## محدودیت‌ها

### در Development:
- API endpoint: `/api/dev/generate-test-data`
- فقط در `NODE_ENV === 'development'` کار می‌کند
- در Production 404 برمی‌گرداند
- **همیشه داده‌های قبلی را قبل از تولید پاک می‌کند**

### در Production:
- کامپوننت `DevToolsPanel` هیچ چیزی رندر نمی‌کند
- API endpoint 404 برمی‌گرداند
- هیچ اروری یا خطایی ایجاد نمی‌کند

## حذف ابزارها

برای حذف این ابزارها از پروژه:

### مرحله 1: حذف کامپوننت
```bash
rm -rf src/components/dev
```

### مرحله 2: حذف API endpoint
```bash
rm -rf src/app/api/dev
```

### مرحله 3: حذف import از Dashboard
در فایل `src/app/page.tsx`، خط زیر را حذف کنید:
```typescript
import { DevToolsPanel } from '@/components/dev/dev-tools-panel';
```

و خط زیر را حذف کنید:
```tsx
<DevToolsPanel />
```

### مرحله 4: حذف authApiDelete (اختیاری)
اگر از این تابع در جای دیگری استفاده نمی‌کنید، می‌توانید آن را از `src/lib/api.ts` حذف کنید.

## نکات مهم

1. **حذف خودکار داده‌ها**:
   - به صورت پیش‌فرض، هر بار که داده تولید می‌کنید، داده‌های قبلی پاک می‌شوند
   - این برای جلوگیری از مشکل داده‌های قدیمی و کش شده است

2. **نرخ تکمیل تصادفی**:
   - داده‌های تولید شده نرخ تکمیل تصادفی (حدود 70%) دارند
   - هر هفته نرخ متفاوتی خواهد داشت
   - هر بار که تولید می‌کنید، نرخ‌ها متفاوت خواهند بود

3. **متن‌های نمونه**:
   - متن‌های تعهد از یک لیست از 10 جمله نمونه انتخاب می‌شوند
   - متن‌های دلایل بازتاب از دو لیست (تکمیل شده / تکمیل نشده) انتخاب می‌شوند

4. **امنیت**:
   - این endpoint نیاز به احراز هویت دارد
   - هر کاربر فقط می‌تواند داده‌های خودش را ببیند و حذف کند
   - در Production کاملاً غیرفعال است

## تست گزارش هفتگی

بعد از تولید داده‌ها:

1. به صفحه `/reports` بروید
2. گزارش هفتگی AI را ببینید
3. با داده‌های مختلف (7، 14، 28 روز) تست کنید
4. نتایج متفاوت را مشاهده کنید

## تغییرات جدید (نسخه 2.0)

### ✅ حذف خودکار داده‌های قبلی
- به صورت پیش‌فرض، هر بار که داده تولید می‌کنید، داده‌های قبلی پاک می‌شوند
- این باعث می‌شود همیشه داده‌های تمیز و جدید داشته باشید
- پیام نشان می‌دهد که چند تعهد و بازتاب حذف شده است

### ✅ UI بهبود یافته
- دکمه‌های جدید با متن "1 هفته جدید"، "2 هفته جدید"، "4 هفته جدید"
- دکمه "پاک کردن" در بالای پنل برای حذف دستی داده‌ها
- نمایش بهتر اطلاعات هفته‌ها

### ✅ API Parameters جدید
- `clearBefore` (boolean): تعیین می‌کند آیا قبل از تولید داده‌ها پاک شوند یا نه
  - به صورت پیش‌فرض: `true`
  - برای حفظ داده‌های قبلی: `false`

## مثال استفاده از API

### با پاک کردن داده‌های قبلی (پیش‌فرض)
```bash
curl -X POST http://localhost:3000/api/dev/generate-test-data \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"days": 14}'
```

### بدون پاک کردن داده‌های قبلی
```bash
curl -X POST http://localhost:3000/api/dev/generate-test-data \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"days": 14, "clearBefore": false}'
```

## مشکلات رایج

### مشکل: ابزارهای توسعه نمایش داده نمی‌شوند
- **راه‌حل**: مطمئن شوید `NODE_ENV` روی `development` تنظیم شده است

### مشکل: API 404 برمی‌گرداند
- **راه‌حل**: در محیط Production هستید. برای تست به Development برگردید

### مشکل: داده‌های قبلی پاک نمی‌شوند
- **راه‌حل**: این رفتار نرمال است. در development، داده‌های قبلی همیشه پاک می‌شوند.
